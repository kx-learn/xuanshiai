> 文档版本：v1.0

## 一、服务基础信息

### 1.1 技术栈

- Python 3.11+
- FastAPI
- Pydantic Settings / Pydantic Schema
- SQLAlchemy AsyncSession
- MySQL 8 + `aiomysql`
- PyMySQL：数据库初始化脚本使用
- JWT：Access Token 和 Refresh Token
- Fernet：身份证等敏感信息加密
- Redis：配置已预留，当前认证验证码逻辑仍使用进程内存储

### 1.2 统一前缀

除根路径外，业务接口统一使用：

```text
/api/v1
```

交互式接口文档：

```text
GET /docs
GET /redoc
```

### 1.3 通用认证方式

需要登录的接口使用：

```http
Authorization: Bearer <access_token>
```

Access Token 中只包含用户 ID、会话 ID、Token 类型和过期时间，不包含手机号、身份证号等敏感信息。

## 四、用户认证与账号安全

### 4.1 发送短信验证码

```http
POST /api/v1/auth/sms/send
```

权限：无需登录。

请求体：

```json
{
  "phone": "13800138000",
  "purpose": "login"
}
```

字段说明：

- `phone`：11 位大陆手机号。
- `purpose`：`login` 登录验证码，或 `bind_phone` 绑定手机号验证码。

业务规则：

- 验证码有效期 5 分钟。
- 同一手机号 60 秒内不能重复发送。
- 同一手机号同一用途每日最多发送 10 次。
- 新验证码会使旧验证码失效。
- 当前 `SMS_PROVIDER=disabled` 时返回 `503`，不会发送真实短信。

响应 `202`：

```json
{
  "message": "验证码已发送",
  "expires_in": 300,
  "retry_after": 60
}
```

### 4.2 手机号验证码登录

```http
POST /api/v1/auth/phone/login
```

权限：无需登录。

请求体：

```json
{
  "phone": "13800138000",
  "purpose": "login",
  "code": "123456",
  "device_id": "device-1",
  "platform": "wechat-mini-program",
  "app_version": "1.0.0"
}
```

业务规则：

- `code` 必须为 6 位数字。
- 未注册手机号会自动创建用户。
- 已冻结或注销账号不能登录。
- 登录时自动补齐普通用户角色。
- 记录登录日志和设备信息。

### 4.3 微信登录

```http
POST /api/v1/auth/wechat/login
```

权限：无需登录。

请求体：

```json
{
  "code": "wx.login-return-code",
  "nickname": "用户昵称",
  "avatar": "https://example.com/avatar.jpg",
  "device_id": "device-1",
  "platform": "wechat-mini-program",
  "app_version": "1.0.0"
}
```

业务规则：

- 使用微信 `jscode2session` 换取 `openid`。
- 同一 `openid` 映射到同一内部用户 ID。
- 首次微信登录自动创建用户。
- 未绑定手机号时返回 `need_bind_phone=true`。
- 微信 AppID 和 AppSecret 必须通过环境变量配置。

### 4.4 绑定手机号

```http
POST /api/v1/auth/bind-phone
```

权限：登录用户。

请求体：

```json
{
  "phone": "13800138000",
  "purpose": "bind_phone",
  "code": "123456",
  "device_id": "device-1",
  "platform": "wechat-mini-program",
  "app_version": "1.0.0"
}
```

业务规则：

- 必须使用 `bind_phone` 用途的验证码。
- 一个手机号只能绑定一个账号。
- 已绑定其他账号时返回 `409`，不会覆盖原绑定。

### 4.5 刷新登录令牌

```http
POST /api/v1/auth/refresh
```

权限：无需 Access Token，但需要有效 Refresh Token。

请求体：

```json
{
  "refresh_token": "<refresh_token>"
}
```

业务规则：

- Refresh Token 有效期默认 30 天。
- 刷新成功后旧 Refresh Token 立即撤销。
- 新会话会继承设备信息。

### 4.6 退出当前设备

```http
POST /api/v1/auth/logout
```

权限：携带当前 Access Token。

用途：撤销当前会话，接口幂等返回 `204`。

### 4.7 退出所有设备

```http
POST /api/v1/auth/logout-all
```

权限：登录用户。

用途：撤销当前用户所有有效会话。

### 4.8 获取当前用户信息

```http
GET /api/v1/auth/me
```

权限：登录用户。

响应字段：

- `id`：用户 ID。
- `phone_masked`：脱敏手机号。
- `nickname`：昵称。
- `avatar`：头像。
- `status`：账号状态。
- `phone_verified`：手机号是否已验证。
- `realname_status`：实名认证状态。
- `need_bind_phone`：是否需要绑定手机号。

### 4.9 查询当前协议版本

```http
GET /api/v1/auth/agreements
```

权限：无需登录。

用途：返回当前发布的协议类型和版本，不返回协议正文。

响应示例：

```json
{
  "user_service": {"type": "user_service", "version": "v1"},
  "privacy_policy": {"type": "privacy_policy", "version": "v1"},
  "safety_pledge": {"type": "safety_pledge", "version": "v1"},
  "community_rules": {"type": "community_rules", "version": "v1"}
}
```

### 4.10 签署用户协议

```http
POST /api/v1/auth/agreements/accept
```

权限：登录用户。

请求体：

```json
{
  "agreement_type": "safety_pledge",
  "version": "v1",
  "content_hash": "<sha256>",
  "scene": "profile"
}
```

业务规则：

- 只允许签署当前发布版本。
- 记录用户、协议版本、时间、IP、设备和场景。
- 重复签署同一用户协议版本具有幂等性。

### 4.11 提交实名认证

```http
POST /api/v1/auth/realname
```

权限：登录用户。

请求体：

```json
{
  "real_name": "张三",
  "id_card": "110101199001011234"
}
```

业务规则：

- 校验姓名长度和身份证格式。
- 身份证对应年龄必须年满 18 周岁。
- 身份证原文加密保存。
- 身份证哈希用于去重。
- 响应只返回身份证脱敏值。
- 当前提交后进入审核中，不伪造第三方认证成功。

响应示例：

```json
{
  "status": 1,
  "id_card_masked": "1101**********1234"
}
```

## 五、注册身份与红娘申请

### 5.1 查询注册身份选项

```http
GET /api/v1/registration/intents
```

权限：无需登录。

当前选项：

| 值 | 名称 | 说明 |
|---|---|---|
| `self_match` | 自己找 | 以本人交友和婚恋匹配为主要目的 |
| `parent_match` | 父母帮找 | 由本人授权父母参与资料和匹配流程 |
| `companion` | 找搭子 | 以兴趣、活动和同城搭子为主要目的 |

### 5.2 提交注册身份

```http
PUT /api/v1/auth/registration-intent
```

权限：登录用户。

请求体：

```json
{
  "intent_type": "self_match",
  "source": "register"
}
```

业务规则：

- 接口具有幂等性。
- 当前版本允许重新选择身份。
- 选择结果保存为用户注册意图，不混入账号状态或隐私状态。

### 5.3 查询当前注册身份

```http
GET /api/v1/auth/registration-intent
```

权限：登录用户。

未选择时返回 `null`。

### 5.4 查询红娘申请类型

```http
GET /api/v1/matchmaker/application-types
```

权限：无需登录。

当前类型：

| 值 | 名称 |
|---|---|
| `promoter` | 推广红娘 |
| `partner` | 合伙人招募 |
| `service_matchmaker` | 服务红娘 |

### 5.5 提交红娘申请

```http
POST /api/v1/matchmaker/applications
```

权限：登录用户，且必须绑定手机号并完成实名认证。

请求体：

```json
{
  "application_type": "service_matchmaker",
  "real_name": "张三",
  "phone": "13800138000",
  "intro": "有多年婚恋咨询和沟通经验",
  "cert_images": ["/storage/application/cert-1.jpg"]
}
```

业务规则：

- 同一用户可以分别申请不同类型。
- 同一类型存在待审核、已通过或已暂停申请时不能重复提交。
- 被驳回的申请可以重新提交。
- 申请状态：`0`待审核、`1`通过、`2`驳回、`3`暂停。

### 5.6 查询我的红娘申请

```http
GET /api/v1/matchmaker/applications/mine
```

权限：登录用户。

用途：返回当前用户提交的全部红娘和合伙人申请，手机号以脱敏形式返回。

### 5.7 管理员审核红娘申请

```http
PATCH /api/v1/admin/matchmaker/applications/{application_id}
```

权限：必须拥有有效 `admin` 角色。

请求体：

```json
{
  "status": 1,
  "fail_reason": null
}
```

状态说明：

- `1`：通过，同时授予对应平台角色。
- `2`：驳回，必须填写 `fail_reason`。
- `3`：暂停，必须填写 `fail_reason`。

## 六、用户资料接口

### 6.1 获取个人资料

```http
GET /api/v1/users/me/profile
```

权限：登录用户。

返回性别、出生日期、服务端计算年龄、身高、职业、行业、学历、收入、地区编码、自我介绍、兴趣标签、性格标签和资料完整度。

### 6.2 修改个人资料

```http
PATCH /api/v1/users/me/profile
```

权限：登录用户。

支持字段：

- `gender`：1 男，2 女；首次提交后不能自行修改。
- `birthday`：出生日期，必须年满 18 周岁。
- `height`：100~250 厘米。
- `occupation`、`industry`：最长 128 个字符。
- `education_level`：1~8。
- `income`：0~1,000,000。
- 省、市、区地区编码。
- `self_intro`：最长 1000 个字符。
- `interest_tags`、`personality_tags`：标签内容不能为空，单个标签最长 32 个字符。

修改后服务端重新计算资料完整度。

### 6.3 查询资料完整度

```http
GET /api/v1/users/me/completion
```

权限：登录用户。

返回：

- `score`：0~100 的完整度分数。
- `missing_items`：未完成项提示。
- `can_browse`：是否达到浏览门槛。
- `can_apply`：是否达到申请门槛。
- `can_chat`：是否达到聊天门槛。

当前门槛：

- 分数达到 40：允许浏览。
- 分数达到 80 且实名认证通过：允许申请和聊天。

## 七、通用错误响应

### 7.1 错误格式

FastAPI 默认错误格式：

```json
{
  "detail": "错误说明"
}
```

参数校验错误可能返回：

```json
{
  "detail": [
    {
      "type": "string_too_long",
      "loc": ["body", "code"],
      "msg": "字段长度不符合要求",
      "input": "..."
    }
  ]
}
```

### 7.2 常见状态码

| 状态码 | 含义 |
|---:|---|
| `200` | 请求成功 |
| `201` | 资源创建成功 |
| `202` | 请求已接受，例如验证码发送 |
| `204` | 请求成功且无响应正文，例如退出登录 |
| `400` | 业务参数错误、验证码错误或过期 |
| `401` | 未登录、Token 无效或会话失效 |
| `403` | 账号不可用、未满足认证门槛或权限不足 |
| `404` | 资源不存在 |
| `409` | 资源或业务状态冲突 |
| `422` | Pydantic 参数校验失败 |
| `429` | 频率限制或验证码错误锁定 |
| `503` | 外部服务未配置或暂时不可用 |

## 八、已实现的业务能力

### 8.1 会话和认证

- 微信登录凭证换取 `openid`。
- 手机号验证码登录和自动注册。
- 微信账号绑定手机号。
- Access Token / Refresh Token。
- Refresh Token 轮换。
- 当前设备退出和全设备退出。
- 最多有效设备会话数限制。
- 登录会话撤销。
- 登录日志记录。

### 8.2 验证码安全

- 验证码哈希后存储。
- 5 分钟过期。
- 60 秒发送间隔。
- 每日发送次数限制。
- 错误 3 次后锁定 15 分钟。
- 验证码成功校验后立即失效。
- 登录验证码和绑定验证码用途隔离。

### 8.3 身份与资料

- 注册意图保存：自己找、父母帮找、找搭子。
- 普通用户角色自动补齐。
- 红娘申请类型区分：推广红娘、合伙人、服务红娘。
- 管理员审核通过后授予对应角色。
- 手机号和身份证脱敏返回。
- 身份证原文加密保存、哈希去重。
- 成年人校验。
- 资料完整度重新计算。

## 十、当前外部依赖和运行限制

### 10.1 微信登录

需要配置：

```env
WECHAT_APP_ID=真实小程序AppID
WECHAT_APP_SECRET=真实小程序密钥
```

未配置时微信登录返回 `503`。

### 10.2 短信验证码

当前默认配置：

```env
SMS_PROVIDER=disabled
```

此状态下不会发送真实短信，发送接口返回 `503`。项目当前还没有具体短信供应商适配器。

### 10.3 数据库

需要 MySQL 8 和 `aiomysql`。数据库表由：

```powershell
python database_setup_marriage.py
```

初始化脚本创建和补齐。

### 10.4 Redis

配置项已经存在，但当前业务代码尚未将验证码限流切换到 Redis。多实例生产部署前必须完成 Redis 限流接入，否则进程内验证码状态无法跨实例共享。
