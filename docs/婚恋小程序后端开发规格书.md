## 二、数据库 ER 图（核心实体关系）

```
用户 (User) ──┬── 1:1 ── 个人资料 (Profile)
              ├── 1:N ── 动态 (Post)
              ├── 1:N ── 点赞/收藏 (Interaction)
              ├── 1:N ── 认识申请 (FriendRequest)
              ├── 1:N ── 私信 (ChatMessage)
              ├── 1:N ── 纸飞机 (PaperAirplane)
              ├── 1:N ── 活动报名 (ActivitySignup)
              ├── 1:N ── 积分记录 (PointLog)
              ├── 1:N ── 会员订单 (MembershipOrder)
              └── N:N ── 双向关注 (Follow)

用户角色：普通用户 | 推广红娘 | 服务红娘 | 合伙人 | 管理员

会员等级：无 | VIP | SVIP
```

```python
# models/user.py
from sqlalchemy import Column, String, Boolean, DateTime, Enum as SAEnum, Text
from sqlalchemy.orm import relationship
import enum

class UserRole(str, enum.Enum):
    USER = "user"                    # 普通用户
    PROMOTER = "promoter"            # 推广红娘（付费入驻）
    MATCHMAKER = "matchmaker"        # 服务红娘（审核入驻）
    PARTNER = "partner"              # 合伙人
    ADMIN = "admin"                  # 管理员

class UserStatus(str, enum.Enum):
    ACTIVE = "active"                # 正常
    PAUSED = "paused"                # 暂停服务
    MATCHED = "matched"              # 已脱单
    BANNED = "banned"                # 封禁
    DELETED = "deleted"              # 已注销

class User(Base):
    __tablename__ = "users"

    id = Column(UUID, primary_key=True, default=uuid4)
    phone = Column(String(20), unique=True, nullable=True)     # 手机号（微信登录后绑定）
    wechat_openid = Column(String(64), unique=True, nullable=True)  # 微信 openid
    password_hash = Column(String(256), nullable=True)          # 密码（手机号登录用）
    nickname = Column(String(50), nullable=False)

    role = Column(SAEnum(UserRole), default=UserRole.USER)
    status = Column(SAEnum(UserStatus), default=UserStatus.ACTIVE)
    vip_level = Column(String(10), default="none")             # none | vip | svip
    vip_expire_at = Column(DateTime, nullable=True)

    # 积分
    points = Column(Integer, default=0)

    # 注册时间
    created_at = Column(DateTime, default=datetime.utcnow)
    last_login_at = Column(DateTime, nullable=True)

    # 软删除
    deleted_at = Column(DateTime, nullable=True)

    # 关系
    profile = relationship("Profile", back_populates="user", uselist=False)
    posts = relationship("Post", back_populates="author")
    sent_requests = relationship("FriendRequest", foreign_keys="FriendRequest.from_user_id")
    received_requests = relationship("FriendRequest", foreign_keys="FriendRequest.to_user_id")
    point_logs = relationship("PointLog", back_populates="user")
```

## 五、业务规则汇总

### 5.1 访客浏览规则
```
免费用户：每日浏览 20 次完整资料
VIP 用户：无限制
合拍度 > 80% 名片：免费用户可完整查看 5 次，超出后部分隐藏
无痕浏览：VIP 可开启，访问不留访客记录
对方拉黑：返回 403，无法访问
```

### 5.2 认识申请规则
```
普通用户：每日免费 3 次
VIP 用户：每日免费 10 次
申请有效期：48 小时（超时自动过期）
对方同意：扣除1次机会 + 开通私信
对方拒绝：全额退款（如已付）
同一用户不能重复申请（已申请则提示）
```

### 5.3 合拍度算法
```
合拍度 = 基础匹配(30%) + 兴趣重合(30%) + MBTI互补(20%) + 活跃度(10%) + 认证加分(10%)

基础匹配：性别+年龄+婚姻+城市+职业+学历+收入 → 0-100分
兴趣重合：兴趣标签交集/并集 → 0-100分
MBTI互补：I/E互补 + N/S互补 → 加成
活跃度：近7天有登录 + 发过动态 + 签到 → 加成
认证：实名+10，学历+10，房产+10（封顶）
```

### 5.4 AI 合拍度匹配（情感实验室）
```
入口：消息 Tab → 情感实验室
若技术支持：跳转 MBTI/星座/生肖测试 → 生成结果 → AI 合拍度匹配
若技术不支持：用户手动输入 MBTI/星座 → 基础合拍度分析
合拍度 0-60：普通匹配
合拍度 60-80：良好匹配（显示原因）
合拍度 80-100：高度匹配（重点推荐）
```

### 5.5 每日定时任务（Cron）
```
00:00 — 重置所有用户今日免费申请次数
00:05 — 检查 VIP/置顶 过期，到期自动降级
01:00 — 统计昨日 DAU/新注册/付费金额
06:00 — 推送早安消息（活跃用户）
08:00 — 推送活动报名提醒（已关注活动）
10:00 — 推送新人推荐（给未匹配用户）
```

### 5.6 隐私控制规则
```
交友限制：开启后仅认证用户可发申请
无痕浏览：VIP专属，开启后访问不留痕
隐藏敏感信息：隐藏学校/公司全称
在线状态：可隐藏
黑名单：双向拉黑，无法互相访问/发申请/私信
状态切换：公开展示/委托红娘/完全私密/暂停服务/已脱单
```

## 七、技术实现要点

### 7.1 文件上传
```
头像/相册：MinIO 或腾讯云 COS，路径 /avatars/{user_id}/{uuid}.jpg
动态图片：/posts/{user_id}/{uuid}.jpg，最多 9 张，WebP 压缩
视频：/videos/{user_id}/{uuid}.mp4，限制 30 秒
签到二维码：服务端生成，含 user_id + activity_id + token
```

### 7.2 推荐缓存策略
```
用户每次访问首页：先查 Redis 缓存推荐列表（key: recommend:{user_id}）
缓存命中率 > 80% 时直接返回
用户点击「换一批」：清空缓存，重新计算
置顶用户：单独 Redis Sorted Set，score=置顶到期时间戳
```

### 7.3 WebSocket（私信实时）
```
路径：/ws/chat/{room_id}?token={jwt}
心跳：每 30 秒 ping
断线重连：前端自动重连，最多重试 3 次
消息投递：写库 → 发 WebSocket → 存储离线消息
```

### 7.4 安全措施
```
所有密码：bcrypt 加密
敏感信息（身份证/学历证明）：加密存储，接口返回时脱敏
防刷：同一 IP 申请次数限制（Redis 计数）
防爬：请求频率限制（rate_limit 中间件）
JWT：RS256 或 HS256，7 天有效期
短信验证码：5 分钟有效，错误 3 次锁定 15 分钟
```
