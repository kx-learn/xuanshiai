## 一、页面基础与全局配置

### 1.1 全局底部导航栏（5 个一级 Tab）
> 每个页面含 AI 合拍度悬浮图标

#### Tab ① 首页（自救专区）

- **推荐**（单人名片）
- **广场**（用户卡片流）
  - 卡片上方有置顶推荐标题栏
  - 每三个名片下方弹出一个会员专区
- **会员专区**
- **筛选搜索框**
- **浏览记录**（小图标）
- **下方按键**（喜欢、申请认识、分享）
- **点击头像** → 进入个人主页

#### Tab ② 社区

- **顶部状态栏**
  - **喜欢**——关注或收藏的人的动态
  - **同城**——支持添加城市，展示合拍的人
  - **发现**（Banner 轮播）
    - 话题标签
    - 线下活动
    - 纸飞机（匿名话题）
      - 纸飞机（匿名话题）
      - 话题（全部话题、近期热门、更多话题）
      - 线下活动
- **用户动态**——发布文字/图片/打卡，支持评论、点赞、分享、关注

#### Tab ③ 牵线

1. **一对一情感咨询**
   - 【服务红娘】——付费专业红娘，1v1 定制牵线
   - 【热心红娘】——红娘排行榜，按介绍朋友数量排序
   - **AI 红娘**——情感问题咨询
   - **底部通用栏**——返回首页、【我要做红娘】

2. **私人定制**
   - 红娘一对一高端婚恋方案

#### Tab ④ 消息

- **申请消息**——查看发出和收到的认识申请
- **纸飞机**——跳转社区纸飞机页面
- **ta 的动态**——跳转社区喜欢页面
- **情感实验室**——人格测试 / MBTI 输入，AI 合拍度匹配
- **聊天记录**

#### Tab ⑤ 我的（个人中心）

**1. 顶部基础状态**

- 我的爆灯（我给谁爆灯 / 谁给我爆灯）

**3. 个人属性与资料深度设置**

## 架构要点总结

### 用户操作主路径
```
注册身份选择 → 基础资料填写 → 底部导航(5 Tab) → 核心交互
                                                         ├─ 首页：推荐/广场名片流 → 喜欢/申请认识
                                                         ├─ 社区：浏览动态/同城 → 互动
                                                         ├─ 牵线：AI红娘 / 服务红娘 / 定制方案
                                                         ├─ 消息：聊天/申请/情感实验室
                                                         └─ 我的：资料/设置/任务/充值
```

### 核心设计特征
| 特征 | 说明 |
|------|------|
| **AI 贯穿** | 每个页面悬浮 AI 合拍度图标；情感实验室人格/MBTI 匹配；AI 红娘 |
| **红娘模式** | 服务红娘(付费1v1) + 热心红娘(排行榜) + 红娘招聘/合伙人 |
| **变现点** | 会员充值、认识申请次数、爆灯功能、私人定制方案 |
| **身份灵活** | 自己找/父母帮找/找搭子，覆盖不同用户群 |
| **轻社区** | 动态发布、同城、纸飞机匿名话题、线下活动 |
| **认证体系** | 实名、学历、房产、婚姻状态、单身承诺多维度认证 |
